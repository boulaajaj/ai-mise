# Deployment model

How AI-Mise gets onto a person's machine, and which platforms we target when.
Decision status: working doc; promote load-bearing choices to ADRs as they harden.

## Target order

1. **Claude Code — now.** The richest file-based harness primitives (skills,
   CLAUDE.md, hooks, permissions, plugins) and the runtime Amine uses daily.
   Everything in Phases 0–6 targets it via `control-plane/adapters/claude-code/`.
2. **Codex CLI / AGENTS.md — later.** Second adapter once the Arduino pilot
   passes; the neutral workspace format (ADR-0001/0002) makes this an adapter
   task, not a rewrite.
3. **OpenHarness — later.** Third target candidate (prior-art.md); proves
   portability against a non-Anthropic runtime.
4. **ChatGPT — not now.** Workspace Agents are enterprise-tied and not
   file-based; there is no clean projection target today. Revisit when their
   surface can consume file-defined instructions/skills. The claims/views/
   memory layers are plain markdown, so nothing is lost by waiting.

## What "installed" means (three pieces)

| Piece | Lives at | Installed how |
|---|---|---|
| **Builder plugin** (skills: inspector, later gateway/retrospective; hooks; scripts) | user-level Claude Code (`~/.claude/` plugin or skills dir) | git clone + install script (later: plugin marketplace) |
| **Control plane** (policy, schemas, validators, approval store) | its own directory outside any workspace, e.g. `~/ai-mise/control-plane/` | same install script; OS-permission hardening applied here |
| **Workspaces** (one per project/profession) | wherever the user keeps projects | generated by the Builder through the gateway |

Uninstall/upgrade: the plugin and control plane are replaceable; workspaces
are plain files + git and survive everything (the portability promise).

## v0 Personal Preview (the "test it on my Claude" slice)

The earliest slice worth putting on Amine's machine daily, ahead of full
Phase 2–5 completion:

1. **Inspector skill** — works after Phase 1 polish (usable in rough form now:
   point Claude Code at `skills/inspector/`).
2. **Mind map** — views generated as an Obsidian-compatible vault (issue #22
   subset): plain markdown + wikilinks; open the folder in Obsidian, graph
   view is the mind map.
3. **Mini CI** — a PostToolUse/pre-commit hook running the existing validators
   on every change (the CI/CD seed from issue #24's comment).
4. **Self-review lite** — end-of-session retrospective in shadow mode:
   observations + proposals to a file, nothing applied (ADR-0003 compliant).

Exit: Amine uses it on a real project for a week and files what annoyed him.
That week of friction is the best requirements document Phase 2–5 can get.
